SAE 5.01 – Concevoir, réaliser et présenter une solution technique

Cette SAE avait pour objectif de concevoir, déployer et documenter une infrastructure de machines virtuelles permettant aux étudiants d'accéder à distance à des environnements de travaux pratiques via un navigateur web.

Objectifs pédagogiques

  • Déployer un hyperviseur (Proxmox) pour héberger plusieurs machines virtuelles thématiques.
  • Mettre en place un accès distant aux machines via une interface web sécurisée.
  • Documenter l’ensemble du déploiement et de la configuration des services utilisés.
  • Tester la robustesse de l’infrastructure et mettre en œuvre une supervision de base.

Organisation du projet

En trinôme, nous avons installé Proxmox sur un serveur physique avec plusieurs interfaces réseau, puis créé différentes machines virtuelles (Linux, Windows, outils réseaux, etc.). Pour permettre l’accès distant, nous avons intégré Apache Guacamole qui permet la prise en main des VM via un simple navigateur web.

Travail réalisé

  • Installation de Proxmox et configuration des ressources matérielles (CPU, RAM, bridge réseau...)
  • Déploiement de VM templates adaptées aux différents TP (Linux, réseau, outils d’analyse...)
  • Déploiement et configuration d’Apache Guacamole pour un accès sécurisé par navigateur
  • Tests de connexion, création de comptes utilisateurs et vérification de la stabilité

Limites et éléments non finalisés

Faute de temps, certaines fonctionnalités prévues initialement n’ont pas pu être finalisées :

  • Implémentation complète d’une supervision centralisée des VM
  • Automatisation complète des déploiements (scripts, playbooks Ansible)
  • Système de gestion des utilisateurs et journalisation avancée

Compétences mobilisées

  • Virtualisation avec Proxmox
  • Réseaux (bridge, VLANs, routage)
  • Déploiement web avec Apache Guacamole
  • Documentation technique claire et structurée

Bilan personnel

Ce projet m’a permis de comprendre les enjeux du déploiement d’une infrastructure mutualisée pour l’enseignement. J’ai renforcé mes compétences en virtualisation, en réseau, et en accès distant sécurisé.

SAÉ 5.02 - Cycle de vie d’un projet informatique

Cette SAE avait pour objectif de mettre en pratique les principes de gestion de versions avec Git dans le cadre d’un mini-projet en binôme. Le but était de rédiger une documentation en Markdown en respectant un cycle de vie structuré, avec plusieurs branches de développement.

Objectifs pédagogiques

  • Appliquer les commandes Git vues en cours dans un contexte collaboratif.
  • Comprendre la structuration d’un projet logiciel versionné.
  • Apprendre à produire une documentation technique versionnée et maintenable.

Organisation du projet

Le projet se déroulait en plusieurs étapes :

  1. Création d’un dépôt Git contenant un README.md et un CHANGELOG.md vide sur la branche master.
  2. Ajout d’un répertoire doc/ contenant :
    • Commandes.md – Documentation des commandes Git vues en cours.
    • Nouveaute.md – Présentation de fonctionnalités avancées non couvertes en cours (hooks, alias, .gitignore).
  3. Utilisation d’un modèle de branches structuré :
    • Une branche par membre (ex : dev-nom).
    • Une branche dev commune pour fusionner les contributions.
    • La branche master ne contient que les versions stables avec tag v1, v2…
  4. Création des versions stables en fusionnant les branches dans master puis en ajoutant un tag.

Travail réalisé

Dans mon binôme, j’ai participé à la rédaction du fichier Nouveaute.md en documentant la fonctionnalité suivante :

  • Les hooks Git : j’ai expliqué leur utilité (exécuter des scripts automatiquement lors d’événements Git) et donné un exemple de hook pre-commit qui vérifie le format des messages de commit.

J’ai également contribué à la gestion des versions en effectuant les fusions dans la branche dev et en mettant à jour le fichier CHANGELOG.md à chaque étape importante.

Compétences mobilisées

  • Utilisation avancée de Git et GitLab (branche, merge, conflits, tags).
  • Rédaction technique avec Markdown.
  • Travail collaboratif en binôme et gestion de projet logiciel.

Lien vers le projet

Le dépôt GitLab du projet est disponible à l'adresse suivante : https://gitlab.univ-artois.fr/lucas_wibaux/projet_gitlab.git

Bilan personnel

Cette SAE m'a permis de renforcer ma compréhension de Git et de son utilisation dans un contexte collaboratif. J'ai appris à structurer un projet avec des branches dédiées, à gérer les fusions et à documenter les changements de manière claire. La rédaction en Markdown m'a également aidé à améliorer mes compétences en documentation technique. Ce projet m'a donné une meilleure vision du cycle de vie d'un projet informatique et de l'importance d'une bonne gestion de versions.

SAE 5.ROM – Déploiement d’un service VoIP avec interface Web

Cette SAE visait à intégrer les services de téléphonie IP avec Asterisk et à développer une interface web pour établir des communications VoIP à l’aide de SIP.js. Le projet s’inscrit dans le cadre du parcours Réseaux Opérateurs et Multimédia.

Objectifs pédagogiques

  • Déployer un serveur Asterisk sur Alpine Linux.
  • Configurer des téléphones IP Fanvil et Cisco pour des communications vocales.
  • Mettre en place une interface d’administration via HTTP.
  • Développer une interface SIP.js en Node.js permettant la communication voix (et potentiellement vidéo).

Travaux réalisés

  • Installation d’Asterisk sur Alpine Linux et démarrage du service.
  • Configuration des comptes SIP dans sip.conf et des boîtes vocales dans voicemail.conf.
  • Connexion de téléphones Fanvil (via interface web) et Cisco (via serveur TFTP).
  • Configuration d’une interface web d’administration avec redirection HTTP (port 8088) et création d’un utilisateur dans manager.conf.
  • Mise en place d’un serveur Node.js avec SIP.js et tentative d’échange via WebRTC.

Limites rencontrées

Malgré les efforts investis, l’échange vocal via l’interface SIP.js n’a pas pu être finalisé. Nous avons rencontré plusieurs problèmes techniques, notamment :

  • Conflit de version avec Asterisk, résolu en changeant de version.
  • Problèmes de permissions sur les fichiers de configuration.
  • Communication inopérante entre le serveur SIP.js et le serveur Asterisk.

Faute de temps, il n’a pas été possible d’aboutir à une solution fonctionnelle, malgré des recherches approfondies.

Compétences mobilisées

  • Déploiement de services VoIP avec Asterisk
  • Configuration d’équipements réseau (Fanvil, Cisco)
  • Administration de serveurs HTTP sous Alpine Linux
  • Développement web temps réel avec SIP.js et Node.js
  • Analyse réseau avec Wireshark

Bilan personnel

Cette SAE m’a permis de découvrir en profondeur le fonctionnement d’Asterisk, les contraintes du WebRTC, et les étapes techniques pour intégrer une solution VoIP dans une interface web. Même si le projet n’a pu être finalisé, j’ai renforcé mes compétences en diagnostic réseau et en intégration de services sur serveur léger.

SAE5.Cyber.03 - Assurer la sécurisation et la supervision avancées d'un système d'information

Contexte du projet

Cette SAE visait à mettre en œuvre un système informatique via la suite ELK (Elasticsearch, Logstash, Kibana), permettant d'identifier et analyser des événements issus de différents équipements ou systèmes (Ubuntu, Apache, Cisco, pfSense).

Travaux réalisés

  • Installation et sécurisation d'Elasticsearch et Kibana
  • Ajout de sources : Filebeat pour Apache/Cisco, Metricbeat pour Ubuntu, Elastic Agent pour pfSense
  • Analyse de journaux (logs système, accès web, événements réseau, traces d'attaques)
  • Configuration de dashboards Kibana et création d’alertes simples

Exemple

  • Logs Apache : détection de requêtes (GET/POST), les differents codes réponses http
  • Ubuntu : collecte de métriques système (CPU, RAM, services actifs) avec Metricbeat
  • Cisco : permet de récupérer les logs d'un routeur par exemple un câble qui a été débrancher
  • pfSense : réception et visualisation de logs réseau via Elastic Agent cela permet par exemple de voir si quelqu'un essaye d'accéder à un site bloqué

Compétences développées

  • Déploiement de solutions de supervision
  • Analyse de logs sur différents équipements/systèmes
  • Configuration sécurisée des agents Filebeat et Elastic Agent
  • Création et interprétation de dashboards Kibana

Bilan personnel

J’ai découvert l’importance d’une bonne supervision dans la détection des incidents. J’ai progressé en administration système, gestion de logs et sécurité. Cette SAE m’a conforté dans mon intérêt pour les métiers liés à la cybersécurité.
